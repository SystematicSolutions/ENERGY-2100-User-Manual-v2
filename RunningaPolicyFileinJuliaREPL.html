Running a Policy File in Julia REPL

Executing a Policy File in the Julia REPL

To run a policy file in the Julia REPL, follow the instructions below.

  1. Open VS Code and open the ENERGY 2100 model folder into the workspace:  File\Open Folder.


  1. Start the Julia REPL from the Command Palette:
  • Open the command palette (View\Command Palette) or (Ctrl+Shift+P)
  • Start the Julia REPL (type Julia: Start REPL). 
  • Wait for the julia>  prompt.


  1. Activate the folder as a Julia project environment:
  • Bring in the package manager, activate the project, and instantiate the project to install dependencies:

using Pkg

Pkg.activate("C:\\ModelFolderName")

Pkg.instantiate()



🕮

Example

using Pkg

Pkg.activate("C:\\2020CanadaPoplar")

Pkg.instantiate()





You must use two backslashes (\\) in the folder path. If you use only one, Julia will not find the correct path.


Alternatively, you could use Pkg.activate(".") to activate the current project:

using Pkg

Pkg.activate(".")

Pkg.instantiate()


  • You can verify that you activated the correct Julia environment by checking the package status:

Pkg.status()

If the correct environment was activated, this command will return your desired model folder. 

🛈

The Project.toml and Manifest.toml are present in the model's root directory and define the Julia environment and dependencies required for running ENERGY 2100. The Pkg.instantiate() function ensures all the necessary dependencies for the current project are installed and available.


  1. Load the ENERGY 2100 Julia package

Now that the ENERGY 2100 Julia environment has been activated, load the model's Julia package:

using EnergyModel


  1. Import ENERGY 2100's modules, constants, or functions you want to use. The database and functions required for running a Julia policy file and viewing its variables include the following: 

import EnergyModel.DB

import EnergyModel.ReadDisk

import EnergyModel.Select

import EnergyModel.ReadDiskAndSets

import EnergyModel.Yr


Alternatively, we could import all these structures in one line using the following:

import EnergyModel: DB, ReadDisk, Select, ReadDiskAndSets, Yr



🛈

See section XXX for a more complete list of possible modules, constants, and functions that are available to import from the EnergyModel Project environment.


  1. Load additional packages as desired. For viewing variables in an easy-to-read format and writing variables to CSV files, load the following: 

using DataFrames, DataFramesMeta, CSV


  1. Include your specific policy file in your workspace:

include("Policy/MyPolicyFile.jl")



🕮

Example

include("Policy/Res_PeakSavings.jl")


Including the policy file in your workspace gives you access to the functions defined in the given policy module. 


  1. Execute the policy code by calling its control function:

MyPolicyModule.PolicyControl(DB)



🛈

By convention, the name of the policy's module is the same as the name of the policy file. Also by convention, the control function of each policy is named PolicyControl and accepts the model's database path, DB, as input.



🕮

Example

Res_PeakSavings.PolicyControl(DB)


The PolicyControl(DB) command above will execute the policy. If there are no errors, it will stop with a julia> prompt. If the policy hits errors during execution and you revise the file to fix the error, or if you make any other revisions to the file, you will need to follow the instructions below to update the package with the newer file version.

🛈

Alternative Method of Executing a Policy from the REPL


The EnergyModel package gives you access to all of the variables, sets, databases, functions, and modules that have been defined in the model. To access these functions, you need to know the hierarchy of the calling functions.


EnergyModel.jl is stored in \src and calls all the files that define the structures and functionality of the model. For example, EnergyModel.jl loads all the modules from Policy.jl: include("../Policy/Policy.jl") which loads all the policy modules defined in the reference case. Each of the policy modules define a calling function: PolicyControl(DB), where DB points to the model's HDF5 database. In Julia's REPL, we are able to access any of the modules and functions defined in the EnergyModel package using the hierarchy and dots (.).

Hierarchy:  EnergyModel loads the "Policy" module which loads specific policy modules (such as: Res_PeakSavings) for which each has a controlling function named PolicyControl(DB).

  So, to call the function defined in the Res_PeakSavings.jl policy file, we can type the following into the REPL:

using EnergyModel

EnergyModel.Policy.Res_PeakSavings.PolicyControl(EnergyModel.DB)

Alternatively, we can assign the EnergyModel package a short name and use the short name in the REPL:

using EnergyModel as M

M.Policy.Res_PeakSavings.PolicyControl(M.DB)

Alternatively, we are able to access the database and the policy module directly (without using the dots (".") by importing the database and including the policy file into the REPL, such as:

import EnergyModel.DB

include("Policy/Res_PeakSavings.jl")

Res_PeakSavings.PolicyControl(DB)

Updating the Julia Package After Making File Revisions

If you make changes to the file while in a VS Code session, do the following to reload the Julia project to include the revised files:

  1. Load the Revise package:

Julia code

using Revise


  1. Include your policy module again:

Julia code

include("Policy/MyPolicyModule.jl")


  1. Run the policy's control procedure again:

Julia code

MyPolicyModule.PolicyControl(DB)


Viewing Variables from the Julia REPL

After the policy file has executed successfully, you may want to verify the policy variables have the expected values. There are multiple ways to view the values of variables from the Julia REPL. These different ways include:

  • Assign the variable to a DataFrame and write it to a CSV file;
  • Assign the variable to a DataFrame and view it in the REPL. If the variable is a large array, you may need to create a subset of the DataFrame to be able to view it easily;
  • Add println, @show, or @info statements to your Julia code that write out values to the screen as the file is being executed;
  • Access the interactive interface (HDF5-Explorer-Pro) to view values of variables; or
  • Run an existing or new output .dta file that contains the variable of interest.

The sections below provide instructions for each of these methods.

🛈

The instructions that follow assume you have performed all the steps above to import the required functions and packages into your Julia workspace.

How to Write a Variable to a CSV File

This section provides instructions on writing to CSV files by first assigning the variable to a DataFrame, then using CSV.write to save it to a file.

🛈 A DataFrame holds the variable in an easy-to-view format that include set identifiers. The DataFrame commands used in the instructions below assume the variable is saved on the database (DB).

To write variables to a CSV file from the Julia REPL, follow the instructions below:

  1. Assign the desired variable to a DataFrame:

Julia code

df = ReadDisk(DataFrame, DB, "DBPartition/VariableName")



🕮

Example

df = ReadDisk(DataFrame, DB, "RInput/xPkSav")


  1. Use the CSV.write function to save the DataFrame (df) to a file:

Julia code

CSV.write("output.csv", df)





By default, the .csv file will be saved to the root model directory. If you want to store the .csv files to a separate folder, first create the folder, then revise the command to be as follows:

Julia code

CSV.write("newfolder/output.csv", df)


  1. As an alternative option, use append = true if you want to append data to an existing csv file without overwriting it:  CSV.write("output.csv", df, append=true)



If you don’t want to read any zero values, you can set the skip_zeros keyword argument to true.


df = ReadDisk(DataFrame, DB, "DBPartition/VariableName", skip_zeros = true)




🕮

Example

df = ReadDisk(DataFrame, DB, "RInput/xPkSav", skip_zeros = true)


How to View a Subset of Variable in the REPL using DataFrames

Most variable arrays in ENERGY 2100 are too large to view in their entirety as a DataFrame directly in the REPL. However, you can see a slice of the variable directly in the REPL using the @rsubset macro (from the DataFramesMeta package). The @rsubset macro allows you to filter rows of a DataFrame. 

To view a slice of a variable in the REPL, follow the instructions below:

  1. Save the variable as a DataFrame:

Julia code

df = ReadDisk(DataFrame, DB, "DBPartition/VariableName")



🕮

Example

df = ReadDisk(DataFrame, DB, "RInput/xPkSav")
  1. Create a subset of the DataFrame by selecting specific elements of the variable's sets: 

Julia code

df_subset = @rsubset df begin

:set1 == "SetKey"

:set2 == "SetKey"

...

end





You must select the set elements using set keys rather than set descriptors.



🕮

Example

df_subset = @rsubset df begin

:EC == "SingleFamilyDetached"

:Enduse == "Light"

:Area == "BC"

:Year == 2030

end



To select more than one set element in a given set, use the logical "or" operator: (||):

Julia code

df_subset = @rsubset df begin

:set1 == "Set1Key1" || :set1 == "Set1Key2" || :set1 == "Set1Key3"

:set2 == "Set2Key"

...

end



🕮

Example

df_subset = @rsubset df begin

:EC == "SingleFamilyDetached" || "SingleFamilyAttached" || "MultiFamily"

:Enduse == "Light"

:Area == "BC"

:Year == 2030

end



Alternatively, to select more than one set element in a given set, you can also use the in operator, which acts as a filter to check if elements in a DataFrame column exist within a specified collection:

Julia code

df_subset = @rsubset df begin

:set1 in ["Set1Key1", "Set1Key2", "Set1Key3"]

:set2 == "Set2Key"

...

end



🕮

Example

df_subset = @rsubset df begin

:EC in ["SingleFamilyDetached","SingleFamilyAttached","MultiFamily"]

:Enduse == "Light"

:Area == "BC"

:Year == 2030

end



To select a range of years, use the less than or equal to operator (<=): 

Julia code

df_subset = @rsubset df begin

:set1 in ["SetKey","SetKey","SetKey",...]

:set2 == "SetKey"

...

firstyear <= :Year <= lastyear

end



🕮

Example

df_subset = @rsubset df begin

:EC in ["SingleFamilyDetached","SingleFamilyAttached","MultiFamily"]

:Enduse == "Light"

:Area == "BC"

2028 <= :Year <= 2030

end

Julia code

dfname <enter>

df_subsetname <enter>

How to View a Variable in the REPL using println

To view a variable in the REPL using println, first read the variable and its sets into your workspace, then loop through the variable's sets and use the Julia println as described in the instructions below.

  1. Identify the sets of the variable you want to view. You can find out more about a variable's sets by importing and using the attributes function data_attrs:

import EnergyModel.data_attrs

data_attrs(DB, "DBPartition/VariableName")


Below shows an example of obtaining the attributes for the variable, xPkSav stored on IInput:

🕮

Example

julia> import EnergyModel.data_attrs


julia> data_attrs(DB, "IInput/xPkSav")


Dict{String, Any} with 7 entries:

  "units"             => "MW"

  "doc"               => "Exogenous Peak Savings (MW)"

  "setdesc_locations" => ["IInput/EnduseDS", "IInput/ECDS", "MainDB/AreaDS", "MainDB/YearDS"]     "dims"              => ["Enduse", "EC", "Area", "Year"]

  "size"              => (6, 42, 25, 67)

  "ndims"             => 4

  "type"              => "variable"


From the output of data_attrs, we see that xPkSav is defined by the sets  "Enduse", "EC", "Area", and "Year".


  1. Read the desired variable and sets into your workspace. By importing and using ReadDiskAndSets, you will read in the variable and sets to the names you specify on the left-hand-side of the equation as follows:

import EnergyModel.ReadDiskAndSets

(;VariableName, Set1Name, Set2Name, ...) = ReadDiskAndSets(DB, "DBPartition/VariableName")



🕮

Example

julia> import EnergyModel.ReadDiskAndSets

julia> (;xPkSav,Enduse,EC,Area,Year) = ReadDiskAndSets(DB, "IInput/xPkSav")



  1. Select sets and write desired variable values using println:

🕮

Example

enduses = Select(Enduse,"Heat")

ecs = Select(EC,["PulpPaperMills","IronSteel"])

areas = Select(Area,"BC")

years = collect(Yr(2024):Yr(2026))


for enduse in enduses, ec in ecs, area in areas, year in years

 println("xPkSav(",Enduse[enduse],",",EC[ec],",",Area[area],",",Year[year],")=",

          xPkSav[enduse,ec,area,year])

end
