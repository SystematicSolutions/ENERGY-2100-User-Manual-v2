Setting Up Reference and Scenario Runs

Developing Reference and Scenario Cases


  1. Create Individual Policy Files to Define a Scenario

A set of policy files (Julia scripts that change the values of model variables) are combined and executed together to create scenarios. The reference case is a scenario that includes a specific set of policies that are assumed will be in existence into the future. Alternative or additional policies can be incorporated to define alternative scenarios. Store all policy files in the \Policy folder. 

🛈

See Policy Analysis for instructions on how to develop policy files in ENERGY 2100. 


  1. Define Scenario Files that Include and Call a Set of One or More Policy Files

Once the desired set of policy files are created and added to the \Policy folder, a scenario file is created to include and execute a desired set of policy files together as a scenario. 


A name is assigned to each scenario, such as Ref25 or HighPrice. Then a scenario file is developed as a Julia script file that 1) includes the desired policy files into the model environment  (‘include “PolicyFile1.jl”) then 2) calls the functions created in the policy files ("PolicyFile1.PolicyControl(DB)"). This scenario file is assigned the name of the Scenario, such as Ref25.jl or HighPrice.jl and housed in the \Policy folder.


Scenario files can be created from scratch, or existing files edited to add or remove policies. Some forecast scenarios might include only a couple policy files. Others, such as reference case scenarios, might include over a hundred policies. 


  1. Modify RunE2100Runs.bat Batch File to Execute the Reference Case and/or Scenario Cases

RunE2100.bat assumes that there is a scenario file with the same name as the scenario parameter, which it uses to call the selected policies, execute the model, and compile the results. These files are saved to the /Policy sub-folder along with the included policies.


  1. Sample steps to develop and execute a scenario in ENERGY 2100:


    1. Let's say you have created two new policy files, PolicyFile1.jl and PolicyFile2.jl, saved to the \Policy folder, that you would like to run as a new scenario. See Policy Analysis for instructions on how to create an ENERGY 2100 policy file.


    1. To test the impact of these two policies in the model, create a Julia scenario file that includes these two policy files in the model environment and calls the control functions contained in the Julia policy files. Save the file with the name of the scenario.


The code snippet below shows the contents of a policy test file named PolicyExample.jl that incorporates PolicyFile1.jl and PolicyFile2.jl into the model. 


Sample Scenario File with name, ScenarioName

#

# ScenarioName.jl – Creates a new scenario

#


module Policy


import ...EnergyModel: DB


include("PolicyFile1.jl")

include("PolicyFile2.jl")


function IncorporatePolicies()

  @info "IncorporatePolicies - ScenarioName"


PolicyFile1.PolicyControl(DB)

PolicyFile2.PolicyControl(DB)

end


end

       

☑

Important:  For the above code to work properly, both PolicyFile1.jl and PolicyFile2.jl must contain a function at the bottom of those files named PolicyControl. 


    1. Set up and execute RunE2100Runs.bat from \2020Model by making a call to RunE2100 and passing arguments specifying the name of the new file as the scenario name parameter.


                                         %1     %2       %3                %4        %5       %6       %7        %8       %9

Call   RunE2100    Ref25   Ref25   TestEmpty   2020    2050    Base     Base     Base    All


where:

Argument 1 (%1) - Scenario Name

Argument 2 (%2) - Reference Policy File

Argument 3 (%3) - Test Policy File

Argument 4 (%4) - Beginning Year

Argument 5 (%5) - Ending Year 

Argument 6 (%6) - Reference Case

Argument 7 (%7) - Scenario for zInitial in Access outputs

Argument 8 (%8) - Economic Model Investments Case (see Jeff)

Argument 9 (%9) - Output Files (All, ExcelDTAs, AccessDTAs, or None)
