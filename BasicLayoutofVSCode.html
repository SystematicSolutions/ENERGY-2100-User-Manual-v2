Basic Layout of VS Code

VS Code's interface is divided into six areas:

  • Editor - The main area to edit your files. You can open as many editors as you like side by side vertically and horizontally.
  • Primary Side Bar - Contains different views like the Explorer to assist you while working on your project.
  • Secondary Side Bar - Opposite the Primary Side Bar. By default, contains the Chat view. Drag and drop views from the Primary Side Bar to the Secondary Side Bar to move them.
  • Status Bar - Information about the opened project and the files you edit.
  • Activity Bar - Located on the far left-hand side. Lets you switch between views and gives you additional context-specific indicators, like the number of outgoing changes when Git is enabled. You can change the position of the Activity Bar.
  • Panel - An additional space for views below the editor region. By default, it contains output, debug information, errors and warnings, and an integrated terminal. The Panel can also be moved to the left or right for more vertical space.

Execute code from the console

In VSCode, you can click on the Explorer pane to show the contents on your current folder.

You can now right click inside the explorer pane to drop down a context menu, and choose the “New File…” option to create a new file. Let’s create a new file called script.jl in the root of the folder.

VSCode might prompt you to install the Julia extension when it sees you have created a .jl file, if you haven’t already. Paste the following inside the script.jl file:

println("hello world")

You can run this file by typing the following in the REPL you already had open:

include("script.jl")

Here’s a screenshot of what it looks like on my computer when I do this:

This is a completely valid way to run Julia code for any analysis.

If you want something you can double click on to run the Julia code, you can paste the following1 in a Run.cmd file in the root directory of the Tutorial folder, and double click on it each time to run the code.

Run.cmd

julia --project=@. -i script.jl

Using the Run.cmd approach has a disadvantage in that it can be slow for running every time because Julia will compile your code every time it starts up.

Julia v1.9 will speed up the startup for a lot of packages but just opening a Julia session and leaving it open can be a very productive workflow in Julia today.

Execute code in VSCode’s REPL

While the above two methods are fine, using VSCode can be a much better experience to using Julia. VSCode offers interactive development, debugging, documentation and a whole suite of features.

So let’s try running it using VSCode.

If you haven’t already done this, add the following code to the script.jl file:

println("hello world")

You can now highlight the code with your mouse, right click on that highlighted code and hit Julia: Execute Code in REPL:

The first time you do this, the Julia VSCode extension will open a Julia REPL for you, and this may take a few seconds. The next time onwards, it should be fairly quick.

When this executes successfully, you’ll see a ✓ mark next to the code, and you’ll see the return value or the output of the code in the REPL.

You can also hit Ctrl + Enter OR Shift + Enter to run the code in a Julia REPL.

Hitting Shift + Enter is a keyboard shortcut for the equivalent to copying the code from your text file into the REPL to execute it, and move your cursor to the next block of code. If you want to do the same thing but don’t want to move to the next block, you can use Ctrl + Enter.

Try adding the following lines and executing them.

import Tutorial

Tutorial.greet()

You’ll notice that you didn’t need to activate any environment here in VSCode. That’s because VSCode does it for you.

Help mode

In VSCode, hover-ing your mouse over any function will show the documentation for that function:

You can also Ctrl + Left Click to open a preview of the function definition.

This can be handy to figure out where a function was defined from the call site.

When you are typing code in VSCode, you can hit Ctrl + Space to trigger auto completion.

You can use the arrow keys to navigate the options, and use Enter or Tab to select your choice.

This means even if you have long variable names, you can quickly type them without making typos.

Debugger

There is also a debugger in VSCode: https://www.julia-vscode.org/docs/stable/userguide/debugging/.

If you are coming from C / C++ / Python etc, you should be warned that the Julia Debugger is new and may not have all the features you are looking for. It also currently only works inside VSCode.
