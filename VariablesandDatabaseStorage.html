Variables and Database Storage

Overview of Variables in ENERGY 2100

Variables in ENERGY 2020 are stored as multi-dimensional arrays whose dimensions are sets. The variables are defined and stored on an HDF5 database (named database.hdf5). As an example, the variable, TotDemand, is a multi-dimensional array having 4 dimensions (or sets). Its sets are Fuel, ECC, Area, and Year. 


The code that defines and creates the model variables on the HDF5 database is stored in the \Database folder. By convention, we have partitioned the HDF5 database into segments with variables assigned to a particular segment.


An example of Julia code that creates a commercial variable (CERSM[Enduse,EC,Area,Year]) on the the HDF5 database and assigns it to a partition named CCalDB is shown below:

  

Example 1:  Creating a Variable, CERSM, on HDF5 Database (from CCalDB.jl):


CERSM = CreateVariableInHDF5(db, "CCalDB/CERSM", (:Enduse, :EC, :Area, :Year),

        "Capital Energy Requirement Multiplier (Btu/Btu)","Btu/Btu")


An example of Julia code that creates a summary variable (TotDemand[Fuel, ECC,Area,Year]) on the the HDF5 database and assigns it to a partition named SOutput is shown below:


Example 2:  Creating a Variable, TotDemand, on HDF5 Database (from SOutput.jl):


TotDemand = CreateVariableInHDF5(db,"SOutput/TotDemand", (:Fuel, :ECC, :Area, :Year),"Energy Demand (TBtu/Yr)","TBtu/Yr")
