Executing RunAll and RunModel

Summary of RunAll and RunModel


RunAll.bat - Executes the entire model from reading in input data through creating a reference case. RunAll.bat is housed in \2020Model.


RunAll_Debug.bat - Executes the entire model from reading in input data through creating a reference case, saving the databases at multiple steps in the process to aid in debugging. RunAll_Debug.bat is housed in \2020Model.


RunModel.bat - Executes the model after historical databases are populated, but before data adjustments and calibration. To start the model at RunModel.bat, your model must already have a \Process database created.


Executing a RunAll

A ‘RunAll’ is a shorthanded term for a full execution of the model, starting with the setting of default parameters and the creation of the initial database. RunAll.bat is located in /2020Model and can be called without any other parameters to execute a full run. Optionally, RunAll_Debug.bat in /2020Model can be used to execute the same run but with additional databases saved during the model calibration for more finely-tuned review.


A RunAll generally should be executed with any modification to the source code or change to input data. Common changes that require a RunAll include:

  • Changes to the model parameters, such as modifying the values in a set.
  • Adding a new variable to the model database
  • Updates to any of the .dat files that contain input data
  • Changes to the macroeconomic model, including updating the default forecast


Executing RunModel

A ‘RunModel’ describes the sequence of commands that the model after incorporating the input data to calibrate the model, apply adjustments, and produce base case and policy forecasts. This sequence is currently stored in batch file format in RunModel.bat in the /2020Model folder. Double clicking this batch file will execute the model using the initial databases created during a RunAll. 


Opening RunModel.bat in a text editor will show the exact sequence of files called and model execution. There are numerous files called and hundreds of lines of the sequence in current model versions.  Below is a short summary of what is occuring in RunModel going from start to finish:


  1. Opens up the initial database (currently known as ‘Process’) and populates historical variables with values from the data input process
  2. Reads in assumptions for the supply and macroeconomic segments and executes supply and macroeconomic calibration
  3. Reads in assumptions for demand segments and calibrates the demands sectors
  4. Reads in parameters for the electric generation segment and calibrates electric supply
  5. Activates and calibrates endogenous electric prices
  6. Creates and applies emissions coefficients
  7. Applies final calibration and forecast adjustments and creates the starting database for forecast scenarios (‘StartBase’)
  8. Executes the base case
  9. Executes policy scenarios


RunModel should be executed to incorporate any modifications to files that are used in the steps above. By convention, these files are generally located in the /Calibration or /Input/Scripts subfolders


Modifying RunModel


Depending on modeling computer speed, RunModel can take a significant amount of time to re-calibrate the model and produce a new forecast. Often, a file will be adjusted, added, or deleted near the bottom of RunModel.bat. To save execution time in these scenarios, RunModel creates ‘stopping points’, or zipped up databases than can be used to re-run the model calibration at specific points instead of needing to re-execute the entire sequence.


These stopping points can be found in the file as commented out ‘Call UnZip DB’ statements that point to the saved name of the database, located in a subfolder in /2020Model. Use the following steps to shorten RunModel to incorporate a change:


  1. Identify where in the file the change is being made, or where the modified file is being called. 
  2. From this location, move upwards until there is an UnZip statement.
  3. Verify that the subfolder with the name used in the UnZip statement contains a zipped database file matching the version you want to use. Remember that RunAll_Debug creates more stopping points, so make sure that the zip file was created. If it is not created, or has an older date, then search upwards for another UnZip statement
  4. Once the UnZip line is chosen, remove the comment (‘rem’) from in front of the line. Then delete all lines above until the UnZip statement is on the first line of the file.
  5. Save the edited batch file to 2020Model using a different name than ‘RunModel.bat’. This file can be considered temporary and deleted after use.
