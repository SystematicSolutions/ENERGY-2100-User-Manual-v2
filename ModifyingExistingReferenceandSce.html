Modifying Existing Reference and Scenario Cases

To modify an existing reference or scenario case, follow the instructions below:


    1. Add new policy files to the \Policy folder to be included in an existing scenario, as appropriate.

🛈

See Policy Analysis for instructions on how to develop policy files in ENERGY 2100. 


    1. Modify the existing scenario file that incorporates policies into the scenario. This scenario file has the same name as the reference or scenario case name, such as Ref25.jl, Ref25A.jl, or HighPrice.jl, and does two things:  includes the policy files in the scope of the model with an "include" statement, and calls the Julia functions defined in each policy file. The function in each policy file that calls the policy file routines is, by  convention, the last function listed in the policy file and is named "PolicyControl".


To add or remove policies from an existing reference or scenario case, modify the scenario file that incorporates the policies into the model (ex:  Ref25.jl, HighPrice.jl), by following the steps below:

      1. Delete or add “include” statements to reflect desired policies in the top portion of the file:

include(“PolicyName.jl")


      1. Within the IncorporatePolicies() function, remove or add calls to each desired policy's PolicyControl function:  

PolicyName.PolicyControl(DB)

🛈

See Developing Reference and Scenario Cases for more detail on the structure of scenario files. 


    1. Set up and execute RunE2100Runs.bat from \2020Model to see the impact of incorporating the policy changes you made. See Executing Reference and Scenario Cases for instructions on setting up RunE2100Runs.bat.


The code sample below illustrates a scenario file that includes a set of policies as part of a Ref25 reference case:


       Sample Scenario File: Ref25.jl:

#

# Ref25.jl - List of policies included in reference case scenario

#


module Policy


import ...EnergyModel: DB


include("PatchFsPOCXTrans.jl")

include("SetAsReferenceCase.jl")

include("Elec_Exports_Patch.jl")

include("Ammonia_Exports.jl")

include("Hydrogen_Prices.jl")


include("ElectricDeliveryCharge_Switch.jl")


#

# Biofuels and Solar

#

include("KPIA_Biofuels_Prov.jl")

include("LCFS_BC.jl")


#

# Residential and Commercial Building Code

#

include("Com_BldgStdPolicy.jl")

include("Res_BldgStdPolicy.jl")


#

# QC Process Retrofit

#

include("QC_EcoPerf_Com.jl")


#

# Residential and Commercial Equipment Standards

#

include("StdEq_Com_U19.jl")

include("StdEq_Res_U19.jl")


...



function IncorporatePolicies()

  @info "IncorporatePolicies - Ref25"


  PatchFsPOCXTrans.PolicyControl(DB)

  SetAsReferenceCase.PolicyControl(DB)

  Elec_Exports_Patch.PolicyControl(DB)

  Ammonia_Exports.PolicyControl(DB)

  Hydrogen_Prices.PolicyControl(DB)


  ElectricDeliveryCharge_Switch.PolicyControl(DB)


  #

  # Biofuels and Solar

  #

  KPIA_Biofuels_Prov.PolicyControl(DB)

  LCFS_BC.PolicyControl(DB)


  #

  # Residential and Commercial Building Code

  #

  Com_BldgStdPolicy.PolicyControl(DB)

  Res_BldgStdPolicy.PolicyControl(DB)


  #

  # QC Process Retrofit

  #

  QC_EcoPerf_Com.PolicyControl(DB)


  #

  # Residential and Commercial Equipment Standards

  #

  StdEq_Com_U19.PolicyControl(DB)

  StdEq_Res_U19.PolicyControl(DB)


  ...


end


end
